- **5 种新精灵素材入库**（贴图由哥哥生成）：

## [0.24.4] - 2026-07-17

> UI架构复用重构（商店/PC仓库/消息框/按钮样式统一为共享组件）+ 清理town/village死代码

### 重构
- **共享UI组件**：新增 `scripts/ui/ShopPanel.gd`（商店面板）、`scripts/ui/PcBoxList.gd`（PC仓库列表）、`scripts/ui/ui_style.gd`（按钮样式辅助），overworld_scene 与此前各自维护一份相同代码的场景统一改为调用共享组件，消除约300行重复
- **DialogBubble 通用化**：扩展外观自定义参数（颜色/尺寸/位置/圆角），battle_scene 的消息框改用它，不再自建一套二层描边框
- **特性数值体系统一**：编辑器的特性数值输入从任意 1-100% 改为对齐游戏已有能力等级公式（`MonDB._stage_mult`）的段/阶体系——能力等级类效果用"段"（-6~6，对应1.5×~4×），固定倍率类效果用"阶"（-5~5，每阶20%），避免出现两套互不对齐的数值系统；编辑器内实时显示换算后的百分比

- **town_scene.gd / village_scene.gd 死代码**：`main.gd` 的场景路由从 260708 起就把"village"/"town"请求统一重定向到 overworld，这两个脚本已无法通过任何路径被实际执行（含旧存档兼容路径），予以删除；`青木村.tscn`/`碧溪镇.tscn`（仍作为地图素材被 大世界.tscn 引用）根节点的悬空脚本引用一并清空

### 修复
- **特性顺序矫正**：8只精灵（烈火猫/蛹甲/贝丽斯/元英/智敏/浅浅丘/酷姐/绵绵羊）的主/隐藏特性强弱颠倒，隐藏特性反而弱于主特性，现已对调

- **编辑器技能面板**：效果类型下拉切换后"概率%"/"数值%"输入框错位（`.form-group` 的 flex 布局被行内 `display:block` 覆盖），已改回 `flex`
- **编辑器技能面板**：效果下拉的事件监听器错误绑定 `this` 导致联动失效，改用箭头函数修复
- **编辑器底栏**：新增数据汇总显示（精灵/技能/特性/道具总数），加载与保存后自动刷新

- **进化链家族视图**：精灵图鉴的"进化链"从只显示自己能进化成什么，改成完整家族视图——自动往上溯源到祖先、往下延伸到底（分叉即停），每只精灵一张小卡片，属性 + 六维种族值迷你条一并显示，方便设计种族值时直接对照上一形态；当前精灵高亮，点祖先/后代卡片可直接跳转（`tools/editor/src/components/type-badge.js` 的 `contrastTextColor` 复用于卡片配色）
- **学习技能列表改版**：从单列长表格改成两列网格卡片，每张卡片带属性/物特变分类/威力标签；技能名、特性 1/2 都换成可搜索输入框（对齐旧编辑器 `_ability_autocomplete` 的体验），不用再在原生下拉里一条条翻
- **精灵列表属性/品阶筛选**：侧边栏搜索框下补回旧编辑器有的属性/品阶筛选下拉（`mon_filter_t1`/`mon_filter_tier` 的对应实现）
- **布局重排**：精灵图预览+基本信息合并到左栏（不再空出大片留白），种族值+遭遇地移到右栏；图鉴描述和属性克制改为并排显示；种族值数值输入加上中文标签（体力/物攻/物防/特攻/特防/速度，中英并列）
- **NPC 立绘预览填满显示**：人物立绘改成按预览框实际高度自适应缩放，不再被 160px 上限卡住导致框内大片空白
- **打包版启动脚本**：新增 `tools/launch_tauri_editor.bat`，直接启动已打包的正式版可执行文件

### 修复
- **徽章颜色主题适配**：属性/学习技能标签徽章改成纯色底 + 按底色亮度自动选黑/白字，取代"半透明底+同色字"的旧方案——后者文字对比度依赖页面主题背景色，日间模式下雷（黄）、夜间模式下暗/格这类偏暗色系会看不清，纯色底徽章两种主题下都稳定可读
- **保存时 JSON 字段被强制按字母序重排**：Rust 后端 `serde_json` 依赖没开 `preserve_order` 特性，默认用 `BTreeMap` 存储对象字段，导致编辑器每次保存都会把 species/npcs/moves 等 JSON 文件的字段顺序打乱成字母序（内容本身没错，但 git diff 会显示整个文件改动，没法正常 review）；加上该 feature 后保存会保留字段原有顺序
- **打包报 `wix314-binaries.zip` 下载超时**：`tauri build` 默认要打 Windows MSI 安装包，需要联网下载 WiX 工具链，网络/代理不稳时会直接失败；改为关闭 bundle（`tauri.conf.json` 的 `bundle.active=false`），只编译可执行文件，不再依赖这个下载
- **`build_tauri_editor.bat`/`run_tauri_editor.bat` 缺少依赖安装步骤**：首次运行 `node_modules` 不存在导致 `command not found: tauri`，现在两个脚本都会在缺少 `node_modules` 时自动先跑 `bun install`
- **NPC 队伍旧格式字段清理**：删掉了"NPC队伍（旧格式）"这块 UI——真实数据里没有一个角色用这个格式（`party` 字段），游戏代码也完全不读它，是 Tauri 编辑器新增时没对应到实际场景的死字段
- **`tools/editor/dist/`（Vite 构建产物）误入 git 追踪**：漏加进 `.gitignore`，导致每次本地构建验证都会把这个应该被忽略的目录标记为改动；已补上忽略规则并取消追踪
